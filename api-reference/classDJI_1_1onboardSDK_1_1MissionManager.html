Detailed Description

MissionManager class for chaining/managing missions.

Constructor & Destructor Documentation

MissionManager::MissionManager ( Vehicle *  vehiclePtr = 0)
Note

Currently the implementation will not affect any porting and refactoring Mission can be called as simple as vehicle->missionManager->waypt/hotpt->func()

MissionManager exists because: It can maintain flexibility and scalability for future application that might need more missions in one flight ex. first use waypt mission to navigate to point A, B, and C At point C, execute a hot point around it After hotpt mission, do a waypt mission to point D, E, and F.

Detail: This manager has two fixed-size container for waypt and hotpt User only needs to call missionManager->init(_,_,_) the manager will instantiate the object and keep a ptr to the latest one At the same time, there will be a counter for protection in case the user is trying to access object that does not exist

Future work:

  1. might need a smarter container for removing mission
  2. might need a smart mechanism to schedule missions
  3. combine the push data event callback to schedule missions
  4. be creative
